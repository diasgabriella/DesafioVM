    <h1>Automatizando a Implantação de WordPress na Azure com Terraform</h1>

    <p>Este guia descreve como automatizar a implantação de um site WordPress na Azure utilizando o Terraform. O processo envolve a configuração de uma máquina virtual (VM), instalação do Docker, e execução de containers para o WordPress e MySQL, garantindo que o banco de dados seja preservado em atualizações.</p>

    <h2>Instruções de Implantação</h2>
    
    <h3>1. Preparação do Ambiente</h3>
    <p>Primeiro, faça o clone deste repositório para sua máquina local utilizando o Git:</p>
    <pre><code>git clone https://github.com/seu-usuario/seu-repositorio.git</code></pre>

    <h3>2. Inicializando o Terraform</h3>
    <p>Dentro do diretório do projeto, abra seu terminal ou prompt de comando e execute os seguintes passos:</p>
    <ul>
        <li><code>terraform init</code>: Configura o ambiente de trabalho do Terraform e baixa os provedores necessários.</li>
        <li><code>terraform plan</code>: Gera um plano de execução, exibindo o que será criado ou alterado na infraestrutura.</li>
        <li><code>terraform apply</code>: Aplica o plano de execução e provisiona os recursos na Azure.</li>
    </ul>

    <h3>3. Acessando o WordPress</h3>
    <p>Após a conclusão da execução do <code>terraform apply</code>, será exibido o endereço IP da VM onde o WordPress está hospedado. Abra esse endereço em um navegador para acessar seu site WordPress.</p>
    <div class="note">
        <p><strong>Aviso:</strong> Aguarde cerca de 30 segundos após a implantação para garantir que os containers do Docker estejam completamente inicializados antes de acessar o site.</p>
    </div>

    <h3>4. Limpeza dos Recursos</h3>
    <p>Se você não precisar mais dos recursos provisionados, poderá removê-los facilmente executando o seguinte comando:</p>
    <pre><code>terraform destroy</code></pre>
    <p>Este comando desprovisionará todos os recursos criados pelo Terraform, revertendo a infraestrutura para o estado original.</p>

    <h2>Configurações Importantes</h2>
    <ul>
        <li><strong>Grupo de Recursos:</strong> Defina o nome do grupo de recursos a ser criado na Azure.</li>
        <li><strong>Localização:</strong> Especifique a região da Azure onde os recursos serão implantados.</li>
        <li><strong>Usuário Administrador:</strong> Informe o nome de usuário para acessar a VM.</li>
        <li><strong>Senha do Administrador:</strong> Defina uma senha forte para o acesso seguro à VM.</li>
    </ul>

    <h2>Considerações Finais</h2>
    <p>Certifique-se de configurar corretamente suas credenciais de acesso à Azure no arquivo <code>provider.tf</code> antes de iniciar o processo de implantação.</p>
